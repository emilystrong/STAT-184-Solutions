Exercise 7.1

The operations listed involve the following corresponding functions:

  1. Reduction; takes the values of a numeric variable and returns a scalar.

  2. Neither; does not return a value for each case, and does not return a scalar.

  3. Reduction; see section 7.3 in the textbook.

  4. If we are given a set of years and want to determine whether or not each one is a leap year, then this involves a Transformation function. If we are given only one year and want to determine whether or not it is a leap year, then this inolves neither type of function, since the input is not a variable.

  5. If we are given a set of dates and want to determine whether or not each one is a legal holiday, then this involves a Transformation function. If we are given only one date and want to determine whether or not it is a legal holiday, then this inolves neither type of function, since the input is not a variable.

  6. Reduction; takes the values of a numeric variable and returns a scalar.

  7. If we are given a set of dates and want to determine the day of the week for each one, then this involves a Transformation function. If we are given only one date and want to determine what day of the week it is, then this inolves neither type of function, since the input is not a variable.

  8. Reduction; takes a set of dates and returns a scalar (the range, in this case).

Exercise 7.6

Part (a)

In the following scatterplot, each glyph (i.e.each dot) represents one level of the State variable in the ZipGeography data. The horizontal position of a glyph represents the total land area for a given state, and its vertical position represents the total population in that state.

data("ZipGeography")
ZipGeography %>%
  group_by(State) %>%
  summarise(State_Land_Area=sum(LandArea, na.rm=TRUE),
            State_Population=sum(Population, na.rm=TRUE)) %>%
  ggplot(aes(y=State_Population, x=State_Land_Area)) +
  geom_point() +
  theme_bw()

Part (b)

The following graph is a choropleth map showing the population of each state. Each glyph is a state on the map. Lighter-colored states have higher populations than darker-colored states.

geoSummary <- ZipGeography %>%
  group_by(State) %>%
  summarise(State_Population = sum(Population, na.rm = TRUE))
USMap(data=geoSummary, key = "State", fill = State_Population)

Part (c)

The following graph is a choropleth map showing the population per unit area of each state. Each glyph is a state on the map. Lighter-colored states have higher population-to-area ratios than darker-colored states.

geoSummary = ZipGeography %>%
  group_by(State) %>%
  summarise(State_Land_Area = sum(LandArea, na.rm = TRUE),
            State_Population = sum(Population, na.rm = TRUE))
geoSummary = na.omit(geoSummary) %>%
  group_by(State) %>%
  summarise(Population_Density = State_Population/State_Land_Area) %>%
  arrange(desc(State))
# removing rows in which "State" is not name of US state, so USMap() will work
geoSummary = geoSummary[-c(which(geoSummary$State==""),
                           which(geoSummary$State=="District of Columbia")),]
USMap(data=geoSummary, key="State", fill=Population_Density)

OR

ZipGeography %>%
  group_by(State) %>%
  summarise(State_Land_Area = sum(LandArea, na.rm = TRUE),
            State_Population = sum(Population, na.rm = TRUE)) %>%
  mutate(Population_Density=State_Population/State_Land_Area) %>%
  mutate(Population_Density=pmin(Population_Density, 500)) %>%
  USMap(key=State, fill=Population_Density)

OR

na.omit((ZipGeography)) %>%
  group_by(State) %>%
  summarise(State_Land_Area = sum(LandArea, na.rm = TRUE),
            State_Population = sum(Population, na.rm = TRUE),
            Population_Density = sum(State_Population)/sum(State_Land_Area)) %>%
  arrange(desc(Population_Density)) %>%
  filter(State!="District of Columbia" & State!="") %>%
  USMap(key="State", fill=Population_Density)
